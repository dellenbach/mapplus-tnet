Konfig-DB steuern

Diese Seite ist nur für Admins gedacht. Der Import ist nicht additiv: je Scope werden Datensätze per UPSERT aktualisiert und verwaiste Einträge gelöscht.

Dadurch lässt sich vermeiden, dass versehentlich aus SLM heraus direkt überschrieben wird. Import, Export und Löschen sind hier bewusst getrennt.

1. Import in die Konfig-DB

Config-Dateien → PostgreSQL
Full-Sync führt UPSERT plus DELETE aus. Das bedeutet: fehlende Einträge in den Quelldateien werden in der DB entfernt.

2. Export aus der Konfig-Store DB

Ausgewählte Kürzel nach core exportieren
Export schreibt die im DB-Bundle gespeicherten Konfigurationsdateien in core/config und core/nls/de und legt vorhandene Dateien als Backup ab.

3. DB-Bundles löschen

Nur ausgewählte Kürzel entfernen
Löschen betrifft nur den Konfig-Store in der DB. Core-Dateien bleiben unverändert, bis ein Export ausgeführt wird.
